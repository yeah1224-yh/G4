# Glossaire

| Terme | Définition |
|-------|------------|
| Cours | Description et les détails associés au cours comme (prérequis) d'un cours incluant son horaire pour une session donnée |
| Avis  | Retour d'expérience d'un étudiant sur un cours suivi |
| Résultats académiques | Resultat des données obtenues après la sessions qui details les informations obtenues (moyenne total, inscrits, échecs)   |
| Charge totale de travail | Estimation du temps et effort requis pour un cours |
| Loi 25 | Loi sur la protection des renseignements personnels au Québec |
| Prérequis | Cours ou conditions obligatoires avant de s'inscrire |
| Diro | Département d'informatique et de recherche opérationnelle de l'UdeM |
| Critère d'acceptation | Condition mesurable pour valider une exigence |
| Personnalisation | Adaptation de l'affichage selon le profil de l'étudiant |
| Profil étudiant | Caractéristiques personnelles (préférences, centre d'intérêt) pour obtenir des résultats de recherche et présentations personnalisés |
| Seuil minimal | Nombre minimum d'avis requis pour afficher des statistiques | 
| Comparaisons | Permet la comparaisons de plusieurs cours pour estimer la charge totale de travail d’une combinaison |
| Soumission Avis| Les étudiants pourrons mettre des avis sur le discord enfin de pouvoir les utiliser plus tard pour la comparaison des cours |
|Filtrage des avis| L'étudiant dois pouvoir filtrer les avis selon différents critères, tels que : la session ou l'année, le volume de travail, le niveau de difficulté perçu |
